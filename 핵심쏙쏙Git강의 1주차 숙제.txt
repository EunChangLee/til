1. 앞으로 git 프로젝트로 만들고 싶은 것
사람들과 멋진 프로젝트를 만들고 싶다.
사람들의 생각들을 모아서 재밌는걸 만들고 나도 더 성장한 나로 만들고 싶다.

2. 1주차 주요 개념 키워드 적어보기
git : 버전 관리 도구
github : 깃 프로젝트를 공유할 수 있는 커뮤니티 사이트
repository : 깃 프로젝트
local repository : 내 컴퓨터의 깃 프로젝트
remote repository : 깃헙같은 공유할 수 있는 공간에 있는 깃 프로젝트
push : 로컬 리포에서 원격 리포로 커밋을 준다.
pull : 원격 리포에서 로컬 리포로 커밋을 가져온다.
commit : 파일의 변경점들을 저장한다.
repository initialize : 깃 프로젝트가 아닌 일반 프로젝트를 깃 프로젝트로 만드는 것.
clone : 깃헙에 있는 프로젝트를 내 컴퓨터에 다운로드 받는다.
tracking : 로컬 리포와 원격 리포를 연결한다. 추적한다.
version 관리 : 무엇이 추가되고 삭제되고 변경되었는지 커밋들을 잘 정리해둔다.
add/staging : 커밋되지 않은 파일들을 스테이지로 올리는걸 말한다.
commit histoy : 커밋의 내역들이다. 커밋한 시간순으로 정리되어있다.
